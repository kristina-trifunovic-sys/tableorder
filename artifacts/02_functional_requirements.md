# functional_requirements

## functional_areas

| area_id | area_name | area_description | related_br_ids |
|---|---|---|---|
| FA-001 | Customer Onboarding | Covers the entry point of the customer experience: QR code scanning, name entry, table-session association, and service mode selection. | [BR-001](01_business_requirements.md#br-001); [BR-003](01_business_requirements.md#br-003); [BR-004](01_business_requirements.md#br-004) |
| FA-002 | Menu Browsing | Covers the display of food and drink menu items with full details — name, description, price, allergen information, and photos — and the real-time running order summary. | [BR-001](01_business_requirements.md#br-001); [BR-002](01_business_requirements.md#br-002); [BR-004](01_business_requirements.md#br-004) |
| FA-003 | Self-Service Order Submission | Covers the complete self-service checkout flow: order review, payment method selection, order submission, unique order number assignment, and cart validation. | [BR-001](01_business_requirements.md#br-001); [BR-002](01_business_requirements.md#br-002); [BR-003](01_business_requirements.md#br-003) |
| FA-004 | Staff-Assisted Service Request | Covers the alternate service path where the customer browses the menu, signals readiness, and the system notifies a staff member to approach the table. | [BR-003](01_business_requirements.md#br-003); [BR-004](01_business_requirements.md#br-004) |
| FA-005 | Order Status Display | Covers the public-facing order status board that shows preparation and pickup-ready states for self-service orders, managed by the admin. | [BR-001](01_business_requirements.md#br-001); [BR-004](01_business_requirements.md#br-004) |
| FA-006 | Menu Management | Covers the protected admin interface for creating, editing, enabling/disabling, and permanently removing menu items. | [BR-002](01_business_requirements.md#br-002); [BR-003](01_business_requirements.md#br-003) |

## requirement_catalog

| fr_id | area_id | requirement_name | requirement_description | primary_actor_ids | input_data | output_data | business_rules |
|---|---|---|---|---|---|---|---|
| FR-001 | [FA-001](#fa-001) | QR Code Session Activation | The system must activate the ordering flow when a customer scans a table-specific QR code, without requiring account creation or login. | [role-001](01_business_requirements.md#role-001) | QR code scan | Active ordering session linked to a specific table | Each QR code must uniquely identify one table; duplicate or unrecognised codes must be rejected gracefully. |
| FR-002 | [FA-001](#fa-001) | Customer Name Entry | The system must prompt the customer to enter their name before any further interaction is allowed. | [role-001](01_business_requirements.md#role-001) | Customer-entered name | Name stored in the active session | Name field cannot be empty; maximum length should be enforced to prevent abuse. |
| FR-003 | [FA-001](#fa-001) | Service Mode Selection | The system must present two mutually exclusive service mode options — self-service and staff-assisted — for the customer to choose from before proceeding. | [role-001](01_business_requirements.md#role-001) | Customer tap/click on a mode | Selected service mode stored in the session | Only one mode may be active per session; the choice locks the subsequent flow. |
| FR-004 | [FA-001](#fa-001) | Table-Session Association | The system must associate the active customer session with the specific table identified by the scanned QR code throughout the entire ordering interaction. | [role-001](01_business_requirements.md#role-001) | QR code table identifier | Session record with table reference | The table identifier must persist for the full lifecycle of the session (from QR scan to order confirmation or staff notification). |
| FR-005 | [FA-002](#fa-002) | Menu Display by Category | The system must display all available food and drink items organised into categories. Items disabled by the admin must not be visible to customers. | [role-001](01_business_requirements.md#role-001) | Active session; menu data from the system | Categorised list of available menu items | Only enabled items are visible; disabled items are suppressed without indication to the customer. |
| FR-006 | [FA-002](#fa-002) | Item Detail Display | The system must display the name, description, price, allergen information, and at least one photo for each menu item. | [role-001](01_business_requirements.md#role-001) | Menu item record | Full item detail card | Allergen information is mandatory and must be prominently displayed; items with missing allergen data must not be published to customers. |
| FR-007 | [FA-002](#fa-002) | Add Item to Order | The system must allow the customer to add one or more items to their active order. | [role-001](01_business_requirements.md#role-001) | Item selection | Updated order cart | Multiple additions of the same item are cumulative; the system increments the quantity. |
| FR-008 | [FA-002](#fa-002) | Adjust Item Quantity | The system must allow the customer to increase or decrease the quantity of each selected item before submitting the order. | [role-001](01_business_requirements.md#role-001) | Quantity change input | Updated order cart with revised quantity | Reducing quantity to zero removes the item from the cart. |
| FR-009 | [FA-002](#fa-002) | Running Order Summary | The system must display a running summary of selected items and the cumulative total amount due, updated in real time as items are added or removed. | [role-001](01_business_requirements.md#role-001) | Active cart state | Live-updated item list and total | The total must reflect all current selections at all times; it must be persistently visible at the bottom of the menu page. |
| FR-010 | [FA-003](#fa-003) | Order Review Before Submission | The system must allow the customer to review their full order summary and total price before confirming submission. | [role-001](01_business_requirements.md#role-001) | Active cart state | Order review screen | No modifications are permitted after the customer confirms the order; the review step is mandatory. |
| FR-011 | [FA-003](#fa-003) | Payment Method Selection | The system must require the customer to select a payment method — card or cash — before the order is submitted. | [role-001](01_business_requirements.md#role-001) | Customer selection (card / cash) | Payment method stored in the order record | Payment method selection is mandatory; the system must prevent submission if none is selected. |
| FR-012 | [FA-003](#fa-003) | Order Submission and Number Assignment | The system must submit the confirmed order and assign a unique order number upon successful submission. | [role-001](01_business_requirements.md#role-001) | Confirmed order and selected payment method | Persisted order record; unique order number | Order numbers must be unique per service session; sequential numbering or a short human-readable code is preferred for display purposes. |
| FR-013 | [FA-003](#fa-003) | Order Confirmation Display | The system must display the assigned order number to the customer immediately after successful order submission. | [role-001](01_business_requirements.md#role-001) | Submitted order record | Confirmation screen with order number | The confirmation screen must also communicate that the order number will appear on the venue's public display when ready. |
| FR-014 | [FA-003](#fa-003) | Empty Cart Validation | The system must prevent order submission if the cart contains no items, and must prompt the customer to add at least one item. | [role-001](01_business_requirements.md#role-001) | Empty cart state | Validation message | Submission is blocked until at least one item is present in the cart. |
| FR-015 | [FA-004](#fa-004) | Menu Display in Staff-Assisted Mode | When the customer selects the staff-assisted service mode, the system must display the full menu so the customer can browse items in advance. | [role-001](01_business_requirements.md#role-001) | Staff-assisted mode selection; menu data | Menu displayed in read-only browsing state | The menu is displayed for browsing only; the customer cannot add items to a cart or submit an order in this mode. |
| FR-016 | [FA-004](#fa-004) | Customer Readiness Signal | The system must provide a clear action for the customer to signal they are ready to be approached by a staff member. | [role-001](01_business_requirements.md#role-001) | Customer action (tap/click "Ready") | Readiness event recorded in the session | The readiness action is a one-time trigger per session; once triggered, it cannot be undone. |
| FR-017 | [FA-004](#fa-004) | Staff Notification | Upon the customer confirming readiness, the system must send a notification to staff indicating that the customer at the identified table is ready to place an order. | [role-001](01_business_requirements.md#role-001); [role-003](01_business_requirements.md#role-003) | Readiness event; table identifier | Notification delivered to staff | The notification must include the table number so the staff member knows which table to approach. |
| FR-018 | [FA-004](#fa-004) | No Payment or Order Number in Staff Mode | The system must not process a payment or generate an order number in the staff-assisted flow; order capture and payment are handled by the staff member directly. | [role-001](01_business_requirements.md#role-001) | Staff-assisted session state | No order record or payment processed | This constraint prevents system state conflicts between self-service and staff-assisted flows. |
| FR-019 | [FA-005](#fa-005) | Public Order Status Display | The system must provide a public-facing display screen accessible within the venue, showing the current status of self-service orders. | [role-001](01_business_requirements.md#role-001); [role-002](01_business_requirements.md#role-002) | Current order records | Public display of order statuses | The display is passively viewed by customers; no authentication is required to view it. |
| FR-020 | [FA-005](#fa-005) | Order Status Tracking | The system must show on the public display which order numbers are currently being prepared and which are ready for pickup. | [role-001](01_business_requirements.md#role-001); [role-002](01_business_requirements.md#role-002) | Order status records | Live order status view | Statuses are: In Preparation and Ready for Pickup. |
| FR-021 | [FA-005](#fa-005) | Admin Mark Order Ready | The system must allow the admin to mark a self-service order as ready for pickup, which immediately updates the public display. | [role-002](01_business_requirements.md#role-002) | Admin action on an order | Updated order status on public display | Only the admin can change order status; the update must be reflected on the public display without delay. |
| FR-022 | [FA-005](#fa-005) | Order Removal from Display | The system must remove an order number from the public display once the customer has collected their order or a configurable timeout has elapsed. | [role-002](01_business_requirements.md#role-002) | Admin dismissal or timeout trigger | Order removed from public display | A configurable timeout ensures the display does not become cluttered with stale entries. |
| FR-023 | [FA-006](#fa-006) | Admin Interface Access Control | The system must provide a protected admin interface accessible only to authorized administrators via username and password authentication. | [role-002](01_business_requirements.md#role-002) | Admin credentials | Authenticated admin session | Access is restricted to a single admin role; failed login attempts should be handled gracefully. |
| FR-024 | [FA-006](#fa-006) | Add Menu Item | The system must allow the admin to add new menu items, specifying name, description, price, category, allergen information, and at least one photo. | [role-002](01_business_requirements.md#role-002) | New item data (name, description, price, category, allergens, photo) | New menu item record persisted and visible to customers if enabled | All mandatory fields (name, price, category, allergen info, photo) must be provided; partial submissions must be rejected. |
| FR-025 | [FA-006](#fa-006) | Edit Menu Item | The system must allow the admin to edit any existing menu item, updating name, description, price, category, allergen information, and photo. | [role-002](01_business_requirements.md#role-002) | Updated item data | Updated menu item record | Changes to price or allergen information take effect immediately for all subsequent customer sessions. |
| FR-026 | [FA-006](#fa-006) | Enable or Disable Menu Item | The system must allow the admin to enable or disable individual menu items; disabled items are hidden from customers without being permanently deleted. | [role-002](01_business_requirements.md#role-002) | Enable/disable toggle for a menu item | Updated item visibility state | Disabling an item does not affect orders already placed that contain that item. |
| FR-027 | [FA-006](#fa-006) | Remove Menu Item Permanently | The system must allow the admin to permanently remove a menu item from the system. | [role-002](01_business_requirements.md#role-002) | Admin delete action on a menu item | Menu item record deleted | Permanent removal should require confirmation to prevent accidental deletion. |
